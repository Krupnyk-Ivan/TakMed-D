## Опис застосунку

**TacMed** — це мобільний застосунок для вивчення тактичної медицини у форматі коротких щоденних сесій (5 хвилин). Застосунок орієнтований на два типи користувачів:

- **Військові** — бойові медики, військовослужбовці
- **Цивільні** — особи, що проходять курс першої домедичної допомоги

### Ключові можливості

- Структуровані навчальні курси з теоретичними матеріалами та чеклістами
- Тести (quiz) для перевірки знань
- Чекліст протоколу MARCH для дій у надзвичайних ситуаціях
- Ігрова механіка: досягнення, бали, серії (streaks)
- AI-чатбот для відповідей на запитання з медицини
- Офлайн-підтримка через локальну базу даних
- Синхронізація даних із хмарним бекендом (Supabase)

## Архітектура

Застосунок побудований за принципами **Clean Architecture** з поділом на три шари в кожному feature-модулі:

```
features/<назва_функції>/
├── data/           # Джерела даних, моделі, реалізації репозиторіїв
├── domain/         # Сутності, інтерфейси репозиторіїв, use cases
└── presentation/   # Сторінки (pages), BLoC-и, локальні віджети
```

**Потік даних:** UI → BLoC → Use Case → Repository → Data Source (Remote / Local)

Для управління залежностями використовується патерн **Service Locator** (GetIt + Injectable з кодогенерацією).

## Встановлення та запуск

### 1. Клонування репозиторію

```bash
git clone <url-репозиторію>
cd takmed
```

### 2. Встановлення залежностей

```bash
flutter pub get
```

### 3. Кодогенерація

```bash
dart run build_runner build --delete-conflicting-outputs
```

> Необхідно виконувати після будь-яких змін у файлах із анотаціями `@freezed`, `@injectable`, `@JsonSerializable`, `@DriftDatabase`, `@RestApi`.

### 4. Налаштування Supabase

Заповніть файл конфігурації з вашими Supabase-кредами:

```dart
// lib/core/config/app_config.dart
const supabaseUrl = 'YOUR_SUPABASE_URL';
const supabaseAnonKey = 'YOUR_SUPABASE_ANON_KEY';
```

### 5. Запуск застосунку

```bash
flutter run
```

Для конкретної платформи:

```bash
flutter run -d android
flutter run -d ios
flutter run -d chrome   # Web
```

### 6. Запуск тестів

```bash
flutter test
```

## Функціональні модулі

### Авторизація (`auth`)
Реєстрація, вхід та скид пароля через Supabase Auth. Токени зберігаються у захищеному сховищі (`flutter_secure_storage`).

### Онбординг (`onboarding`)
Одноразовий екран вибору навчального треку: **Військовий** або **Цивільний**. Вибір зберігається локально та впливає на контент курсів.

### Навчання (`learning`)
- Перелік та детальний перегляд курсів
- Теоретичні уроки з текстовим та медіа-контентом
- Уроки у форматі чекліста (покрокові інструкції)

### Тести (`quiz`)
Запитання з вибором відповіді після проходження уроку. Відображення результатів із поясненнями.

### MARCH-протокол (`march`)
Інтерактивний чекліст за протоколом **MARCH** (Massive hemorrhage, Airway, Respiration, Circulation, Hypothermia) для відпрацювання дій у надзвичайній ситуації.

### Гейміфікація (`gamification`)
- Нарахування балів за завершені уроки та тести
- Серії (streaks) — нагорода за щоденне навчання
- Система досягнень (badges)
- Візуальний прогрес із анімаціями (Lottie, Confetti)

### AI-чатбот (`ai_chat`)
Чат із AI-асистентом для відповідей на запитання з тактичної медицини.

### Профіль (`profile`)
Перегляд та редагування облікового запису, загальна статистика навчання.

## Навігація

Застосунок використовує **Go Router** із захищеними маршрутами (redirect-guards на основі стану `AuthBloc`).

| Маршрут | Екран |
|---|---|
| `/splash` | Екран завантаження |
| `/onboarding` | Вибір треку |
| `/auth` | Вибір способу входу |
| `/auth/login` | Вхід |
| `/auth/sign-up` | Реєстрація |
| `/auth/password-reset` | Скид пароля |
| `/` | Головний екран |
| `/course/:id` | Деталі курсу |
| `/lesson/:id` | Урок |
| `/quiz` | Тест |
| `/quiz/result` | Результати тесту |
| `/march-checklist` | Чекліст MARCH |
| `/ai-chat` | AI-чатбот |
| `/gamification` | Досягнення та прогрес |
| `/profile` | Профіль |
